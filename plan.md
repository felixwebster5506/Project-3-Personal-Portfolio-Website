# Project Plan

## Project Overview
This website will be a personal portfolio showcasing my work history and variety of technical skills.

## Project Objectives
- Create a two page website
- Communicate skills in my portfolio effectively
- Upload project to github

## Technical Risks
- Broken Links between the two pages
- Broken Styling

**Mitigation**
- Open the pages as im working on them
- Navigate between the two pages frequently

## Administrative Risks
- Forgetting to document as im working

**Mitigation**
- Fill out md files intermittently as the project goal becomes more clear

## Management Risks
- Getting stuck on specific website features or design aspects

**Mitigation**
- Create functional HTML first, CSS second


## Environmental Risks
- Unexpected computer issues

**Mitigation**
- Regularly commit changes to github

# Work Breakdown Structure

## Phase 1: Planning
- Decide website topic
- Decide style of the website

## Phase 2: Development
- Create the HTML and CSS for the website

## Phase 3: Testing
- Ensure the two HTML pages are properly connected
- Make sure the styling is consistent

## Phase 4: Documentation
- Write the plan.md
- Write the retrospective.md
- Write the final draft of the scope.md

## Phase 5: Submission
- Push final changes to github
- Upload link to the repository on Canvas
